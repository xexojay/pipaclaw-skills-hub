# Color System Playbook

配色不是“挑一个主色”，而是定义一个可以贯穿整套 deck 的系统。

## 目标

一套可用的 PPT 配色，至少要同时解决：
- 背景与正文对比
- 强调色的节制使用
- 图表色的可读性
- 分隔层级是否清楚
- 不同页面之间是否统一

## Palette Recipes

### `ivory_blue`

适合：
- 正式提案
- 研究提案
- 咨询式商务 deck

结构：
- 暖白底
- 深蓝正文
- 清洁蓝强调
- 很浅的蓝灰辅助面板

效果：
- 专业
- 温和
- 不会显得像默认模板

### `slate_blue`

适合：
- board
- investor
- 偏理性的 evidence deck

结构：
- 冷白或浅灰底
- 深石板正文
- 克制蓝强调
- 边框和表格分隔更明显

效果：
- 更稳
- 更像决策材料

### `ink_blue`

适合：
- 研究报告
- 海外市场分析

结构：
- 纸白底
- 墨蓝正文
- 低饱和蓝青强调
- 少量深色章节条

效果：
- 编辑感
- 研究感

### `midnight_cyan`

适合：
- 发布会
- demo reveal

结构：
- 深色背景
- 高亮浅青强调
- 少量发光感点缀

效果：
- 高对比
- 更像舞台

### `terracotta_editorial`

适合：
- 品牌提案
- campaign
- 外部传播

结构：
- 奶油底
- 深棕正文
- 赤陶色强调
- 沙色面板

效果：
- 更有态度
- 但仍然能读

### `teal_neutral`

适合：
- 培训
- 内部说明
- 轻数据汇报

结构：
- 白底
- 深灰正文
- 青绿色强调
- 冷灰辅助线

效果：
- 清爽
- 高扫描效率

## Rules

- 强调色不要到处用。按钮感、数字重点、章节标、关键线条才用。
- 一页里不要同时有 3 个抢眼色。
- 图表色和 UI 装饰色要分开，避免看起来像海报。
- 深色 deck 只适合少字、高节奏的场景；长文档感 deck 不要硬上深色。
- 如果品牌色过亮或过脏，不要整页铺满，用它做强调即可。

## Chart Color Rule

- 真实业务图表优先 1 主色 + 1 次色 + 中性灰。
- 如果有重点系列，只高亮那一条，其他系列退后。
- 不要为了“丰富”给每根柱子都上不同颜色。
