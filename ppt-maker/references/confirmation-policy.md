# Confirmation Policy

Not every stage needs user interruption. Confirm only when risk, meaning, or delivery compatibility changes materially.

## Required User Confirmations

- `Brief lock`: goal, audience, scene, language, and deck type are clear enough to proceed.
- `Delivery lock`: desktop / WeChat / both is correct when editability matters.
- `Fact lock`: critical numbers/claims and assumption labels are accepted.

Everything else should default to internal team judgment unless the user explicitly asked to review intermediate work.

## Do Not Ask the User to Decide

- card layout vs split layout
- palette tuning inside an already-fit style
- title size hierarchy
- icon style
- page rhythm
- whether to use a timeline, cards, or 2-column comparison when all would preserve the same meaning

These are design execution choices, not user homework.

## Auto-Continue Stages

- Template candidate exploration.
- Story shaping when it stays within the accepted brief.
- Drafting page copy from approved outline.
- Non-critical visual generation (illustrations, separators, iconography).
- Layout refinements that do not change meaning.
- Typography and spacing refinements.
- Page-type selection inside an accepted template family.

## Escalation Triggers (Pause and Ask)

- Conflicting sources for key metrics.
- Missing required materials that block factual slides.
- Large mismatch between brand constraints and chosen style.
- Language ambiguity for bilingual decks.
- Generated visuals that can be misleading for claims/data.
- Two or more genuinely different narrative directions that would lead to different audience conclusions.

## Fast Path

If user says "auto mode":
- collect minimum intake once
- proceed without intermediate approval
- still enforce `Fact lock` before final delivery

Even without explicit "auto mode", prefer this posture by default unless the user asks to approve each step.

## Reveal Policy

- `Draft` is an internal artifact by default.
- Do not send a rough deck merely to prove progress.
- Show the user a reviewed outline if the main risk is narrative or fact coverage.
- Show the user a reviewed near-final deck if the remaining work is mostly polish.

## Confirmation Message Format

Use short, decision-oriented prompts:

- `Decision to confirm`
- `What will change if approved`
- `Default if no response`
- `Risk if skipped`
